# 快速开始指南 - Quick Start Guide

## 🚀 5分钟快速上手

### 第一步：确认Skill已安装

检查你的项目目录结构是否如下：

```
Weibo Trends/
└── .claude/
    └── skills/
        └── weibo-trends-analyzer/
            ├── SKILL.md                          # 核心技能文件
            ├── README.md                         # 说明文档
            ├── dashboard-template-example.html   # 示例模板
            └── QUICKSTART.md                     # 本文件
```

如果是，说明Skill已正确安装！✅

### 第二步：API已内置，无需配置！

**好消息**：Skill已内置天行数据微博热搜API，无需任何配置！

**内置API**：
- 提供商：天行数据 (TianAPI)
- 地址：https://apis.tianapi.com/weibohot/index
- 特点：实时更新、免费使用、稳定可靠

**数据格式**：
```json
{
  "code": 200,
  "msg": "success",
  "result": {
    "list": [
      {
        "hotword": "热搜关键词",
        "hotwordnum": "1234567",
        "hottag": "新/热/荐"
      }
    ]
  }
}
```

如果你有自己的API密钥想使用，可以访问 https://www.tianapi.com/ 注册获取。

### 第三步：调用Skill

在Claude Code中，直接说：

```
分析微博热搜
```

或者更详细的请求：

```
分析今日微博热搜并生成产品创意报告
```

或者指定关注点：

```
分析微博热搜，重点关注可以做成文创产品的话题
```

### 第四步：等待分析完成

Claude会自动：
1. ✅ 从API获取热搜数据
2. ✅ 对每个热搜进行3-5次web搜索
3. ✅ AI分析并生成产品创意
4. ✅ 计算评分（100分制）
5. ✅ 生成可视化HTML报告

### 第五步：查看结果

分析完成后，会生成文件：
- `weibo-trends-analysis-2026-01-11.html` - 在浏览器中打开查看

## 📊 理解评分系统

### 总分 = 100分

| 维度 | 分值 | 关键考量 |
|------|------|----------|
| **可发展度** | 40分 | 市场大吗？能持久吗？技术可行吗？ |
| **有趣度** | 20分 | 够创新吗？够吸引人吗？会传播吗？ |
| **有用度** | 20分 | 实用吗？解决问题吗？受众大吗？ |
| **易生产** | 20分 | 好做吗？材料好找吗？成本合理吗？ |

### 评级标准

- **🏆 优秀 (≥80分)**: 强烈推荐开发，市场潜力大
- **⭐ 良好 (60-79分)**: 可考虑开发，需进一步评估
- **📋 其他 (<60分)**: 观望或需优化

## 🎨 Dashboard功能说明

生成的HTML报告包含：

### 交互功能
- 🔍 **搜索**: 输入关键词快速找到相关产品
- 🎚️ **筛选**: 按评级（优秀/良好/其他）筛选
- 📊 **排序**: 按分数或热搜排名排序
- 🔄 **重置**: 一键清除所有筛选

### 信息展示
- 📈 **头部统计**: 总体数据概览
- 🎯 **产品卡片**: 每个产品的详细信息
- 📊 **可视化评分**: 进度条展示各维度得分
- 📝 **背景研究**: 社交媒体、新闻、用户洞察

## 🔧 故障排除

### 问题：Skill没有触发

**解决方案**:
1. 确认SKILL.md在正确路径
2. 在请求中明确提到"微博热搜"
3. 重启Claude Code

### 问题：API返回错误

**解决方案**:
1. 在浏览器中测试API地址
2. 检查是否需要API密钥
3. 使用测试数据验证Skill功能

### 问题：生成的HTML乱码

**解决方案**:
1. 用支持UTF-8的浏览器打开
2. 右键→编码→UTF-8
3. 使用Chrome或Firefox浏览器

### 问题：搜索结果不相关

**解决方案**:
1. 在请求中指定搜索重点
2. 限制分析的热搜数量
3. 手动提供更多背景信息
